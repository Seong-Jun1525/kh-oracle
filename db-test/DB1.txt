사례1. 
1-1. 사용자 계정을 만들기 위해 일반 사용자 계정인 TEST계정으로 접속하여 계정이 SAMPLE 비밀번호가 1234인 계정을 생성하기 위해 CREATE USER SAMPLE; 를 실행하니 정상적으로 실행이 되지 않았다.
=> 먼저 사용자 계정을 생성하기 위해서는 관리자 계정으로 접속해야 하고 CREATE USER SAMPLE 만 했을 경우 비밀번호가 설정되지 않았기 때문에 정상적으로 실행되지 않은 것이다.

1-2 또한 계정을 생성(CREATE명령만 실행 함)을 하여서 접속하려는 데 접속이 되지 않고 테이블도 생성되지 않았다.
=> 계정을 생성한 후에 데이터베이스 접속 권한과 자원관리를 할 수 있는 권한을 관리자 계정으로부터 부여받지 못했기 때문에 접속이 되지 않고 테이블도 생성되지 않은 것이다.​

사례 2
문제 1. 연산자 우선순위로 인해 OR보다 AND가 먼저 연산되므로 부서코드 부분이 정상적으로 비교되지 않게 된다.
문제 2. 급여가 3백만원 이상(급여 >= 3백만원)이 조건이지만 쿼리문에서는 'SALARY > 3000000' 으로 3백만원 초과로 비교 하고 있으므로 일치하지 않는다.
문제 3. EMAIL에서 '_' 앞에 글자가 3글자인 것이 조건인데 쿼리문에서는 'EMAIL LIKE '____%' 로 작성하여 최소 4글자로 시작하는 것이면 전부 일치하게 되므로 검색 조건과 일치하지 않게 된다.
문제 4. BONUS가 있는 것이 조건이지만 쿼리문에서는 IS NULL 로 인해 BONUS가 NULL 값인 경우 즉 보너스가 없는 경우로 찾는 조건을 주고 있다.
문제 5. 검색 조건으로 남자인 사원을 찾고 있는데 현재 쿼리문에서는 성별관련 조건이 아예 없다.
위의 문제들로 인해 제대로된 결과가 출력되지 않는다.​

============================================================
1-1 해결방안 : 먼저 관리자 계정으로 접속을 해야하고 CREATE USER SAMPLE IDENTIFIED BY '1234'; 를 작성해야한다.
1-2 해결방안 : 먼저 관리자 계정으로 접속한 다음 GRANT문을 사용하여 데이터베이스에 접속할 수 있고 자원을 관리할 수 있는 권한을 부여받아야 한다. GRANT CONNECT, RESOURCE TO SAMPLE; 을 작성하여 데이터베이스 접속 및 자원을 관리할 수 있는 권한을 부여할 수 있다.

2 해결 방안 (통합작성)

SELECT EMP_NAME, EMP_NO, DEPT_CODE, SALARY
FROM EMPLOYEE
WHERE DEPT_CODE IN ('D9', 'D6') AND SALARY >= 3000000 AND BONUS IS NOT NULL
AND SUBSTR(EMP_NO, 8, 1) = '1' AND EMAIL LIKE '___#_%' ESCAPE '#';

=> IN 연산자를 사용해서 DEPT_CODE가 'D9' 이거나 'D6'인 경우 참이되는 조건을 작성 혹은 (DEPT_CODE='D9' OR DEPT_CODE='D6') 식으로 ()로 묶어 줄 수도 있다.
=> SALARY >= 3000000 으로 >= 연산자를 사용하여 3000000원도 포함할 수 있도록 변경
=> EMAIL LIKE '___#_%' ESCAPE '#' 으로 ESCAPE로 나만의 와일드카드 생성하여 문제 해결
=> BONUS IS NOT NULL로 변경하여 BONUS가 NULL값이 아닐 경우 참이 되도록 변경
=> SUBSTR(EMP_NO, 8, 1) = '1'를 작성해서 EMP_NO 컬럼에서 8번째 자리부터 1개의 문자를 추출하여 그 값이 '1'인 경우 참이 되도록 하는 조건 추가 (남자인 경우 주민번호 뒷자리가 1 혹은 3으로 시작한다. 테이블 데이터들은 1인 경우만 있으므로 1만 작성)
